JavaScript Framework: AngularJS

Why Would Someone Use AngularJS?

The dynamic web applications get their structure through AngularJS framework construction. Developers can use HTML to build templates before extending the language syntax to create application components in a simplified and direct way. AngularJS finds its perfect application in developing single-page applications (SPAs) due to its two-way data binding and dependency injection and modular architecture features.

Pros and Cons of AngularJS

Pros:

Two-Way Data Binding: The system performs automatic synchronization between model data and view data to cut down repetitive coding.

Dependency Injection: Testing becomes easier while reusable components find promotion.

Modular Architecture: The approach supports both independent system modules as well as flexible growth.

Community Support: Documentation remains extensive but the developer community responds less actively to Angular 1 since alternative frameworks gain more attention.

Cons:

Performance Issues: Performance bottlenecks appear during two-way data binding operations within large applications.

Steep Learning Curve: The engineer needs to understand scopes as well as directives and dependency injection.

Outdated: The AngularJS maintenance has ended (as of December 2021) yet developers should opt for Angular (version 2 and above) in new project development.

History of AngularJS

AngularJS originated from Misko Hevery and Adam Abrons during 2009 before Google secured ownership of the project. High popularity developed for AngularJS because it brought novel template-based framework combined with two-way data binding. The discontinuation of active AngularJS development did not prevent it from providing baseline concepts which evolved into modern versions of Angular such as Angular (version 2 and higher).


The CLI ng generate command

The command creates both Angular components and generates four corresponding files which include TypeScript and HTML template along with CSS and test features. Four different files appear during creation through this command: TypeScript file, HTML template, CSS file and test file. The following code represents the produced TypeScript file:

        
        import { Component, OnInit } from '@angular/core';
        
        @Component({
          selector: 'app-example',
          templateUrl: './example.component.html',
          styleUrls: ['./example.component.css']
        })
        export class ExampleComponent implements OnInit {
          constructor() { }
        
          ngOnInit() { }
        }
